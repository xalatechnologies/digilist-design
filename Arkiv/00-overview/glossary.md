# Glossary

**Common terms used throughout the Digilist platform documentation.**

---

## A

**Actor Type** - Pricing tier based on organization type (private, business,
sports_club, youth_organization, school, municipality)

**Approval Workflow** - Configurable process for reviewing and approving booking
requests

---

## B

**Blackout Period** - Time period when a listing is unavailable for booking

**Booking** - A reserved time slot for a listing or zone

**Booking Mode** - How bookings are processed (auto_approve, approval_required,
instant_paid, hybrid)

---

## C

**Case Handler** - Municipal staff member who processes booking requests
(Saksbehandler)

**CLAUDE.md** - Documentation file for AI assistants (Claude, Cursor)

---

## D

**Design Tokens** - Semantic values for colors, spacing, typography defined in
`@xalatechnologies/design-tokens`

**Drizzle ORM** - TypeScript ORM used for database access

---

## F

**Feature Flag** - Runtime configuration to enable/disable features per tenant

---

## G

**GDPR** - General Data Protection Regulation (EU data protection law)

---

## I

**ID-porten** - Norwegian national authentication service

**i18n** - Internationalization (localization support)

---

## L

**Leietaker** - Tenant/booker (end user who books resources)

**Listing** - The canonical bookable entity in Digilist (spaces, resources,
events, services). Replaces the legacy "Listing" concept.

---

## M

**Monorepo** - Single repository containing multiple packages and applications

**Multi-Tenant** - Architecture supporting multiple isolated tenants
(municipalities)

---

## O

**Organization** - Entity within a tenant (e.g., sports club, school)

---

## P

**Package** - Shared code module used across multiple applications

**PRD** - Product Requirements Document

**PII** - Personally Identifiable Information

---

## R

**RBAC** - Role-Based Access Control

**RLS** - Row-Level Security (database-level tenant isolation)

**Runbook** - Operational documentation for maintaining a system

---

## S

**SaaS** - Software as a Service

**Scope** - Permission scope (user, org, tenant, saas)

**Shell** - Application shell system for consistent app structure

**SRSD** - Software Requirements Specification Document

---

## T

**Tenant** - A municipality or organization using Digilist

**Token** - See Design Tokens

---
