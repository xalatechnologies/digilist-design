# Listing Detail Sample Data Pack

This section contains realistic sample data for validating the Listing Detail UI.

## No Screen Designs

As specified, this section does not include screen designs. The data is provided for use in testing and validation of the Listing Detail UI components.

## Data Structure

The sample data includes 4 listings covering all booking models:

1. **Space – Conference Room** (TIME_RANGE)
2. **Equipment – Sound System** (QUANTITY)
3. **Space – Sports Hall** (SLOT)
4. **Service – Event Package** (PACKAGE)

Each listing includes complete data for:
- Basic information (name, type, description, location)
- Media (images)
- Facilities
- Additional services
- Rules
- FAQ
- Contact information
- Opening hours
- Booking time policy
- Availability data

## Usage

This data can be used to:
- Test UI components with different booking models
- Validate state handling (loading, empty, disabled)
- Test responsive layouts
- Verify accessibility features
- Demonstrate edge cases
